# V1.9
### Core
1. **Bocha Web Search API 集成**：系统全新接入 Bocha Web Search API，利用互联网实时搜索能力，强化现有的大模型推演模型。

### Feature
1. **洞察与分析模型升级**：所有的视频和爆款公式分析现在会先执行自动网络检索，并主动获取该视频选题对应的实时网络热议焦点和背景，使其产出的洞察不再局限于“视频本身”，而是拥有“环境与社会”维度的宏观解释力。
2. **支持自主配置搜索引擎 API**：在全局设置面板中新增了 Bocha Web Search API Key 的配置模块。
3. **平台洞察支持本地导入**：在模拟投流界面的平台洞察区块，新增了“上传报告”快捷入口，用户可以导入本地的文本分析文档以覆盖当前结果；右侧“确认上传”按钮增加了“上传成功”的反馈特效，以提升用户交互确认感。

### Fixed
1. **修复了模拟投流状态丢失的问题**：在生成沙盘模拟报告和上传图文分析内容后，如果用户切换到其他界面再返回，之前输入的参数、图文素材以及生成的预测报告会被保留，不再丢失。
2. **修复文本文档上传乱码问题**：优化了模拟投流页面平台洞察及其他文本文档上传场景的解析策略。现在系统在读取本地文件时会尝试使用 UTF-8 编码读取，并在遇到问题时自动回退降级使用 GBK 编码进行解析，有效解决了部分在 Windows 平台通过记事本创建的中文文档上传后显示乱码的问题。
3. **修复 PDF/Word 文档解析导致界面卡顿的问题**：对刚加入的“上传报告”快捷入口增加了二进制大文件（如 `.pdf, .doc, .docx`）的格式与体积（大于2MB）拦截处理。当用户强行上传此类不可直接渲染的文档时，系统会给出友好提示和文件载入成功占位，避免了将二进制数据强行作为文本塞入 Markdown 渲染引擎从而导致浏览器卡死的问题。
4. **修复了微博获取评论报错问题**：调整了部分底层获取微博评论时使用的 API 路由协议，替换了已不稳定的 Web V2 接口，从而解决了偶尔出现的 400 Bad Request 请求失败的问题。
5. **修复自动监控图表数据不显示问题**：修复了在“博主信息库”进行博主自动监控或者生成后台监控报告时，因内部数据键名定义错误（返回了 `latest` 而图表组件读取 `current`）导致监控图表中最新视频各项指标数据丢失，仅展示历史平均值柱状图的问题。
6. **修复 B站数据获取不可用的问题**：针对部分 B站接口由于大网拦截或调整导致无征兆返回 HTML 格式响应从而引起 JSON 解析崩溃报 `<doctype...` 错误的情况，替换了 B站用户主页调取的底层链路，并全局加强了反制非 JSON 响应的防护机制，大幅降低加载不可用的情况。
7. **修复小红书无法拉取近期视频（不足两条）的问题**：修复了当用户使用链接而非原生的 userID 保存小红书博主时，因为底层接口仅支持原生 userID 查询导致拉取视频列表空返回的错误情况。系统现在具备了二次反解能力，可以自动从链接获取真实 ID 进行查询。
8. **修复微博全网监控无热点评论样本提供的问题**：修复了在自动监控后台进行博主洞察分析时，因为底层微博接口更新导致原路由与解析结构变动，导致热点评论样本无法被正确提取到的问题，对嵌套层级做出了结构自适应兼容。
9. **深度修复小红书与微博在部分环境下的 400 报错与数据拉取为空的核心问题**：
    - **小红书主页解析 400 错误：** 识别到底层针对分享口令（`share_text`）校验规则收紧后导致了原 WEB API 彻底抛弃对文本协议的支持，引发 400 崩溃。现不仅是主页解析接口，所有涉及小红书的数据拉取（涵盖视频/笔记查询、话题搜索等）均全面升级至 `app_v2` 最新路由，该路由原生兼容且对防爬限制具有极高容灾力。
    - **微博热点评论样本与小红书近期视频丢失（提取为 0 条或空）：** 平台底层由于防特征追踪调整了 JSON 输出嵌套层级。现我们在解析模块植入了基于 Schema 特征检测的递归游标（`Deep Search Array Parser`），大幅增强了长效健壮性，即使未来底层节点名称（如从 `notes` 变为 `note_list` 或嵌套在 `data` 内的任意深度）产生变动，我们依然可以无缝抽取视频列表与热点评论样本，杜绝“该博主近期视频不足两条”的报错。
10. **B节点赞等核心互动数据丢失展示修复**：通过升级 B站视频源的底层抽象，自适应兼容了 B站近期切换 `item.stat` 统计字段嵌套的升级变更，使包含点赞在内的深度互动数据能够在数据看板以及分析模块中完全恢复读取能力。
